# Data Capture Schedule & Roles

## Overview
Two focused data capture sprints split inventory into **legacy stock** (existing shelves/back room) and **new arrivals** (incoming purchase orders). Each sprint includes prep, counting, validation, and backlog review.

## Team roles
- **Project Lead (Owner/Admin):** Approves schedule, monitors progress sheet, resolves blockers.
- **Inventory Captains (2 people):** Execute Quick Add, photo capture, and count verification.
- **Runner (1 person):** Fetch pairs from storage, re-box after photos.
- **QA Reviewer:** Checks data accuracy in the system at the end of each day.

## Daily stand-up checklist (08:45)
1. Confirm device availability and Wi-Fi.
2. Review yesterday's blockers and QA findings.
3. Assign shelves/purchase orders to each captain.
4. Update progress sheet status before starting.

## End-of-day checklist (17:00)
1. Ensure all photos uploaded (check queued uploads).
2. QA reviewer samples 10 items: verify brand, model, price, stock.
3. Update progress sheet with actual counts completed vs. target.
4. Note any follow-ups in the `Notes` column.

## Buffer handling
- If >15% of daily target remains, extend into the next morning and adjust new arrival tasks.
- For missing sizes, log them in progress sheet with status `Recount Needed`.

## Communication
- Use dedicated WhatsApp group `POS Data Capture` for quick updates.
- Project lead posts daily summary and next-day focus.

## Supplies checklist
- Laminated Quick Add card (2 copies)
- Disinfecting wipes, lint rollers
- Portable fan (avoid staff fatigue)
- Extension cord + phone chargers
- Label stickers for boxes awaiting recount
